TaskFlow — AWS Deployment

DynamoDB · Lambda · API Gateway · Cognito  |  Frontend hosted on protozoa.amazon.dev

0 Prerequisites one-time

Install the AWS CLI and SAM CLI, then configure your credentials.

# Install (Windows — Chocolatey)
choco install awscli aws-sam-cli

# Or download installers from:
#   https://aws.amazon.com/cli/
#   https://aws.amazon.com/serverless/sam/

aws configure
# Enter: Access Key ID, Secret Access Key, Region (e.g. us-east-1), output: json

1 Build & Deploy the Stack

Navigate to the AWS/ folder and run SAM.

cd f:\WebDevelopment\TaskFlow\AWS

sam build

sam deploy --guided
# Prompts:
#   Stack name        : taskflow
#   Region            : us-east-1
#   Environment       : prod
#   ExternalFrontendUrl : leave default for now, update in Step 4
#   Save to samconfig : Y
No database setup needed. The DynamoDB table is created automatically by CloudFormation. No schema to run, no seeding required.

3 Upload Frontend to protozoa.amazon.dev

Upload the three files in AWS/frontend/ to your prototype slot via the normal protozoa workflow. Note the GUID assigned to your prototype.

Files to upload:  index.html  app.js  styles.css

4 Set Your Prototype GUID manual

Once you know the GUID, re-deploy with it so Cognito accepts the callback URL.

sam deploy --parameter-overrides ExternalFrontendUrl=https://protozoa.amazon.dev/prototypes/YOUR-GUID
Important: The URL must match exactly — including trailing slash if present — what Cognito shows as the callback URL.

5 User Access

There are two approaches. For an internal Amazon tool, Option A is strongly recommended — employees log in with their existing Amazon credentials and you never manage passwords at all.

Option A — SAML Federation (recommended for Amazon internal tools)

Cognito acts as a broker: it redirects the login button to Amazon's corporate identity provider (Federate / Midway). The IdP authenticates the employee and sends back a SAML assertion. Cognito converts that into a JWT. No user accounts to create or maintain. Any valid Amazon employee account works automatically.

What you need from your internal IT / security team:
The SAML 2.0 metadata URL for Amazon's IdP — typically something like https://idp.amazon.com/saml/metadata or a Federate endpoint. Ask your team's AWS account admin or check the internal SSO documentation.

Add this block to AWS/template.yaml inside Resources:, then re-run sam deploy:

  # In template.yaml → Resources:
  CorporateIdP:
    Type: AWS::Cognito::UserPoolIdentityProvider
    Properties:
      UserPoolId: !Ref UserPool
      ProviderName: AmazonCorporate
      ProviderType: SAML
      ProviderDetails:
        # Replace with the real metadata URL from your IT team
        MetadataURL: 'https://idp.amazon.com/saml/metadata'
        IDPSignout: true
      AttributeMapping:
        # Maps SAML claims → Cognito user attributes
        email: 'http://schemas.xmlsoap.org/ws/2005/05/identity/claims/emailaddress'
        name:  'http://schemas.xmlsoap.org/ws/2005/05/identity/claims/name'

Also update the UserPoolClient in template.yaml to use the IdP instead of Cognito's own login:

  # In UserPoolClient → Properties:
  SupportedIdentityProviders:
    - AmazonCorporate   # remove "COGNITO" so only SSO login is shown

After re-deploying, the Cognito Hosted UI login button will redirect to Amazon's SSO page. Employees authenticate there and are sent back to the app — fully transparent, no Cognito password.

Option B — Manual user creation (small teams / fallback)

Create each user individually. They receive a temporary password and must change it on first login.

aws cognito-idp admin-create-user \
  --user-pool-id       us-east-1_AbCdEfGhI \
  --username           user@amazon.com \
  --temporary-password Temp1234! \
  --message-action     SUPPRESS

6 Teardown (optional)

To remove all AWS resources except the DynamoDB table (which is set to DeletionPolicy: Retain to protect your data):

sam delete --stack-name taskflow

To also delete the table: go to the AWS Console → DynamoDB → Tables → taskflow-prod → Delete.

Cost Estimate

~$0/month at typical usage. All services stay within AWS free tier limits: DynamoDB on-demand (25 GB free), Lambda (1M requests free), API Gateway (1M free), Cognito (50k MAU free).
